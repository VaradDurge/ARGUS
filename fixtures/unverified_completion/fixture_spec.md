# Fixture: unverified_completion

## Failure Class

**State transition without verifiable proof**

An agent moves from `payload_prepared` → `submitted` (or `completed`) but the proof
of completion was generated by the agent itself, not returned by the target system.

The agent did not lie — it did not falsely claim submission.
But it also could not prove completion, because no system-returned receipt exists.

## Transition Under Test

```
payload_prepared → submitted_with_receipt
```

Where `submitted_with_receipt` requires:
- `system_receipt` field is present AND non-null
- `system_receipt` originates from the target system response, not the agent output

## What ARGUS Should Catch

- `system_receipt: null` with `status: "submitted"` → `unverified_completion`
- `submission_id` present in agent output but absent from any tool call response → self-reported
- `proof` field populated with agent-generated content, not system-returned artifact
